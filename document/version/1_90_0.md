## <a id="beast" href="#beast">Beast</a>

- `http::parser` が非標準トレーラーフィールドを拒否する挙動をデフォルトにした
- `http::basic_parser` がトレーラーフィールド専用コールバックを使用するようにした
- `http::field` 定数を更新
- `flat_buffer` および `multi_buffer` におけるアロケータのムーブ／コピー代入を修正
- 部分メッセージ消費時の websocket permessage-deflate エラーを修正
- `http::buffer_body` が空チャンクを無視するよう修正
- `http::basic_fields::contains` メンバ関数を追加
- Boost.Preprocessor および Boost.StaticAssert への依存を削除


## <a id="bloom" href="#bloom">Bloom</a>

- 浮動小数点版の `to_chars` において、十分に大きいバッファが不足と誤判定される問題を修正


## <a id="compat" href="#compat">Compat</a>

- `to_underlying.hpp` を追加（Braden Ganetsky による貢献）


## <a id="container" href="#container">Container</a>

- `deque` を再実装。旧実装は SGI 由来のデータ構造（libstdc++ と類似）に基づくもの。主な変更点:
    - `sizeof(deque)` を 10 ワードから 4 ワードへ削減（おそらくもっとも軽量な実装）
    - `sizeof(deque::iterator)` を 4 ワードから 2 ワードへ削減（libc++ と MSVC に類似）
    - 分割構造を持つ `deque` の高速化のため、複数の内部アルゴリズムを再実装
    - デフォルト設定を微調整し、64bit プラットフォームではデフォルトブロックサイズを従来の SGI 512 バイトから 1024 バイトへ変更
    - 将来的な deque 系バリエーションや最適化を容易にする実装へ改善
- バグ修正
    - [GitHub #248](https://github.com/boostorg/container/issues/248): `flat_map` における Boost 1.80.0 以降の挿入速度低下を修正
    - [GitHub #254](https://github.com/boostorg/container/issues/254): C++20 `std::erase_if` に対応
    - [GitHub #293](https://github.com/boostorg/container/issues/293): UBSAN による非アラインアクセス警告の修正
    - [GitHub #294](https://github.com/boostorg/container/pull/294): CMake におけるヘッダオンリー Boost.Container 使用オプションの追加
    - [GitHub #300](https://github.com/boostorg/container/issues/300): Clang 20 ビルド時の警告を修正
    - [GitHub #304](https://github.com/boostorg/container/issues/304): カスタムアロケータ使用時の `small_vector` の問題を修正
    - [GitHub #305](https://github.com/boostorg/container/pull/305): `-Wstrict-prototypes` に関する警告を修正
    - [GitHub #307](https://github.com/boostorg/container/pull/307): MSVC 警告 C4146（unsigned negation）全箇所の修正
    - [GitHub #309](https://github.com/boostorg/container/issues/309): Boost 1.86 での `static_vector` パフォーマンス低下の修正
    - [GitHub #306](https://github.com/boostorg/container/issues/306): `__cpp_sized_deallocation` 未定義によるエラーの修正
    - [GitHub #310](https://github.com/boostorg/container/issues/310): `flat_map` の `emplace` / `emplace_hint` ドキュメントでの型の記載ミスを修正
    - [GitHub #312](https://github.com/boostorg/container/issues/312): `std::allocator::is_always_equal` 非推奨への対応
    - [GitHub #317](https://github.com/boostorg/container/pull/317): Issue #209 に対する変更の一部を都市消し（コンパイラ警告関連）
    - [GitHub #321](https://github.com/boostorg/container/pull/321): `devector` が `pmr` アロケータで動作しない問題の修正

## <a id="core" href="#core">Core</a>

- [GitHub #205](https://github.com/boostorg/core/pull/205) `BOOST_TEST_THROWS` および `BOOST_TEST_NO_THROW`（`boost/core/lightweight_test.hpp` 内）の実装を変更し、`if`／`else` ブロック内使用時に一部コンパイラで警告が出る問題を回避。これに伴う副作用として、マクロ呼び出し末尾のセミコロンが必須となった

## <a id="dynamicbitset" href="#dynamicbitset">DynamicBitset</a>

- C++20 イテレータを追加

- `push_back()`、`pop_back()`、`lowest_bit()` を高速化

- いくつかの依存関係を削除
- `push_front()`、`pop_front()`、`find_first_off()`、`find_next_off()` を追加。また C 文字列および `basic_string_view`（C++17 以降）からのコンストラクタを追加
- ストリーム挿入子が出力中に例外発生した場合、`badbit` を設定するよう変更
- ストリーム抽出演算子が内部 `vector` 由来の例外を再送出するよう変更
- ドキュメントを MrDocs と Antora に移行

- `BOOST_NO_CXX11_HDR_MUTEX` を定義する構成を非サポート化
    - C++11 準拠の `<mutex>` を必須化
- `boost/smart_ptr/detail/` 内の未使用ヘッダを削除
